## ✨ О проекте

**LoL Meta Analyzer** — это современное desktop-приложение для глубокого анализа патчей League of Legends. Приложение автоматически собирает данные с официальных источников Riot Games, анализирует изменения чемпионов, рун и предметов, а также предоставляет прогнозы влияния изменений на мету игры.

### 🎯 Зачем это нужно?

- **Экономия времени**: Не нужно вручную искать информацию о патчах на разных сайтах
- **Исторический анализ**: Сравнение меты между патчами за последние 20 обновлений
- **Прогнозирование**: Понимание того, как изменения повлияют на силу чемпионов
- **Визуализация данных**: Красивый и интуитивный интерфейс для работы с данными

## 🚀 Возможности

### 📋 Анализ патч-нотов
Автоматический парсинг и структурирование изменений из официальных патч-нотов Riot Games. Все изменения чемпионов, рун и предметов в удобном формате с визуальными индикаторами (усиления/ослабления).

### 📊 Отслеживание меты
Сравнение статистики чемпионов между патчами:
- **Win Rate** — изменение процента побед
- **Pick Rate** — изменение частоты выбора
- **Ban Rate** — изменение частоты банов
- Визуализация трендов и изменений

### 🔮 Прогнозирование
Умный анализ текстовых изменений из патч-нотов для предсказания влияния на мету. Система определяет, какие чемпионы получат усиления или ослабления на основе описания изменений.

### 📚 История изменений
Детальная история изменений для каждого чемпиона, руны или предмета за последние 20 патчей. Удобный поиск и фильтрация по различным критериям.

### 🏆 Тир-лист
Автоматически генерируемый тир-лист чемпионов, рун и предметов на основе количества изменений и их характера (баффы/нерфы).

### 🎨 Современный UI
- **Темная/светлая тема** с автоматическим переключением
- **Адаптивный дизайн** для комфортной работы
- **Интуитивная навигация** между разделами
- **Быстрый поиск** по чемпионам, рунам и предметам

### Технический стек

- **Backend**: Rust (Tauri v2) — быстрый и безопасный бэкенд
- **Frontend**: React 18 + TypeScript — современный и типобезопасный UI
- **Стилизация**: Tailwind CSS — быстрая разработка и кастомизация
- **База данных**: SQLite (sqlx) — легковесная и надежная БД
- **Парсинг**: scraper + reqwest — эффективный парсинг HTML и HTTP-запросы
- **Сборка**: Vite — молниеносная сборка и HMR

## 💻 Разработка

### Предварительные требования

- **Node.js** 18+ ([скачать](https://nodejs.org/))
- **Rust** latest stable ([скачать](https://www.rust-lang.org/tools/install))
- **Git** для клонирования репозитория

### Установка

```bash
# Клонирование репозитория
git clone https://github.com/RaspizDIYs/patch-analyzer.git
cd patch-analyzer

# Установка зависимостей
npm install
```

### Запуск в режиме разработки

```bash
# Запуск dev-сервера с hot-reload
npm run tauri dev
```

Приложение откроется автоматически и будет обновляться при изменении кода.

Обновление локального бандла аугментов с вики (офлайн, перед релизом): из каталога `src-tauri` выполнить `cargo run --bin wiki_augment_fetch` — результат в `src/data/augments-bundled.json`.

### Сборка релиза

```bash
# Сборка всех форматов (MSI + NSIS)
npm run tauri build

# Или отдельно:
npm run tauri:build:msi   # Только MSI
npm run tauri:build:nsis  # Только NSIS
```

Собранные файлы будут находиться в `src-tauri/target/release/bundle/`

## 📖 Документация

### Структура проекта

```
lol-meta-analyzer/
├── src/                          # React фронтенд

│

│   ├── src/
│   │   ├── lib.rs                # Tauri команды и обработчики
│   │   ├── scraper.rs            # Парсер OP.GG и Riot Games
│   │   ├── db.rs                 # SQLite логика и запросы
│   │   ├── analyzer.rs           # Логика сравнения патчей
│   │   └── models.rs             # Структуры данных
│   ├── arch/                     # Локальные ресурсы (иконки рун)

│
├── public/                        # Статические ресурсы
└── .github/workflows/             # CI/CD конфигурация
```

### Основные компоненты

#### Backend (Rust)

- **`scraper.rs`** — парсинг HTML с сайтов Riot Games и OP.GG
- **`db.rs`** — работа с SQLite базой данных, хранение патчей
- **`analyzer.rs`** — сравнение патчей и вычисление изменений меты
- **`lib.rs`** — Tauri команды для взаимодействия с фронтендом

#### Frontend (React)

- **`App.tsx`** — главный компонент с роутингом и состоянием
- **Компоненты страниц**:
  - `PatchReleaseView` — отображение патч-нотов
  - `MetaChangesView` — изменения меты
  - `PredictionsView` — прогнозы влияния изменений
  - `TierListView` — тир-лист чемпионов/рун/предметов
  - `ChampionHistoryView` — история изменений

### Использование API

Приложение использует следующие источники данных:

- **Riot Games Data Dragon** — иконки чемпионов, рун, предметов
- **League of Legends Patch Notes** — официальные патч-ноты
- **OP.GG** — статистика чемпионов (Win Rate, Pick Rate)

## 🎮 Использование

### Первый запуск

1. **Скачайте** и установите приложение из [релизов](https://github.com/RaspizDIYs/patch-analyzer/releases)

Перейдите на главную страницу, чтобы увидеть все изменения из выбранного патча. Изменения сгруппированы по категориям (чемпионы, руны, предметы).

Страница "Сдвиги Меты" показывает сравнение статистики между патчами. Видно, какие чемпионы получили наибольшие изменения Win Rate и Pick Rate.

#### 🔮 Прогнозы
Страница "Прогнозы" показывает предсказания влияния изменений на мету на основе анализа текстовых изменений из патч-нотов.

#### 📚 История изменений
Выберите чемпиона, руну или предмет, чтобы увидеть полную историю изменений за последние 20 патчей.

#### 🏆 Тир-лист
Автоматически генерируемый тир-лист на основе количества и характера изменений.

## 🔧 Настройка

### База данных

Приложение автоматически создает SQLite базу данных `patches.db` в директории приложения при первом запуске. База данных хранит:

- Данные о патчах (версия, дата)
- Изменения чемпионов, рун и предметов
- Статистику меты

### Темы

Приложение поддерживает три режима темы:

- **Светлая** — белые блоки, черный текст
- **Темная** — темные блоки, белый текст
- **Системная** — автоматическое определение на основе настроек ОС

Тема сохраняется в локальном хранилище и применяется при следующем запуске.
